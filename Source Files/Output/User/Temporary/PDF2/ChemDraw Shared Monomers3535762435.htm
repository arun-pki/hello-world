ChemDraw Shared Monomers

ChemDraw can be used as a full featured monomer editing and curation tool. Monomers can be downloaded from web, network or local locations, and written anywhere the user has write permission.
Monomers can be entered de-novo, or imported via CDXML files or from JSON libraries. Once imported, monomers can be edited and moved or copied between libraries.
The ChemDraw Shared Monomers feature is available only in ChemOffice Professional.
To view or change the list of available monomer libraries:
Go to File>Preferences.
The preferences dialog appears. Click on the Monomer Libraries option.

A list of available monomer libraries is displayed along with the following information:
Name: The name of the monomer library.
Access: Specifies whether the corresponding monomer library is read only (Read-only) or editable (Read\Write).
PerkinElmer is a default monomer library provided with ChemDraw and is non-editable (Read-only).
Location: The location of the monomer library.
Sync Status: The status of auto-synchronization. Displays 'Good' when auto-synchronisation is succesful.
Auto- synchronisation of monomer libraries on remote and network locations with the local cached versions happen only when ChemDraw is launched. For more details, "HELM Monomer Library Synchronization" on page 1.
You can order the list of monomer libraries displayed by dragging the monomer library to the desired position. A horizontal line and a blue label indicates the position where the monomer library will be placed. While dragging upwards the monomer library will be placed above the horizontal line and below the horizontal line while dragging downwards.

Adding a Monomer library

You can add a monomer library from three types of locations:
local files
files in a shared network disk
web files over HTTP/HTTPS
When adding monomer libraries, if it has monomers that are already existing in the temporary Document library, those monomers are removed from the temporary Document library and associated with the newly added library.
To add a monomer library from files in a local or shared network location:
Go to File>Preferences.
The preferences dialog appears. Click on the Monomer Libraries option.
Click on the '+' symbol or click " " and select 'Add File...' option.

Browse to the location of the monomer library on the network or local disk, select the library file (*.json) and click OK.
The monomer library is added and is displayed in the Preferences dialog.
When you add a monomer library, it is selected as the datasource type in the HELM Monomer toolbar by default.
To add a monomer library from a HTTP or HTTPS location:
You can also add a monomer library from a web URL (http: or Https:). To do so:
In the Preferences dialog, click " " and select "Add URL..." from the context menu that appears. The Add URL dialog appears.

Specify the monomer library URL and click Save.


The monomer library is added and is displayed in the Preferences dialog.
When you add a monomer library, it is selected as the datasource type in the HELM Monomer toolbar by default.

 

Creating a blank monomer library
You can create a blank monomer library and later add monomers to it using the monomer editors.
To create a blank monomer library:
In the Preferences dialog, click " " and select "Create New File" from the context menu that appears.
Browse to the monomer library location, specify a name and click Save.
The monomer library is added and is displayed in the Preferences dialog.
You can now add monomers to this library by either adding new monomers using the Monomer editor or by choosing to copy, save or move monomers from another library to this using Monomer Viewer / Editor / Importer.

 

Deleting a Monomer library

You can delete a monomer library that is no longer required via the Preferences dialog.
To delete a monomer library:
Go to File>Preferences.
The preferences dialog appears. Click on the Monomer Libraries option.
Select the desired monomer library and click on the '-' symbol.
The selected monomer library is deleted.
If the monomer library being deleted has unsaved monomers in the current document, those monomers are moved to the temporary Document library.
 

Renaming a Monomer library

You can rename any monomer library except the default PerkinElmer library via the Preferences dialog.
You cannot rename a monomer library when auto-sync is in progress.
To rename a monomer library:
Go to File>Preferences.
The preferences dialog appears. Click on the Monomer Libraries option.
Select the desired monomer library to be renamed.
Click on the Name cell to open it in edit mode.
Enter the new name and click anywhere outside the cell.
The monomer library is renamed and the change is reflected in both Preferences table and HELM Monomers toolbar. You can also edit and replace an existing library path (Location) in the same way.
The names ‘PerkinElmer’ and ‘Document’ are reserved library names and cannot be used to name any other libraries.
